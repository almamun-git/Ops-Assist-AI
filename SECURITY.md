# Security Policy

## Supported Versions
Latest commit on `main` is supported.

## Reporting a Vulnerability
Email `almamun.codes@gmail.com` with:
- Component / endpoint
- Steps to reproduce
- Impact assessment
- Suggested mitigation (optional)

Acknowledgment within 48 hours.

## Handling Process
1. Reproduce privately
2. Patch on a protected branch
3. Add tests
4. Document in CHANGELOG

## Scope
- Incident detection logic
- AI analysis input sanitization
- Future auth / authorization layers
- Data validation and environment configuration

## Out of Scope
- Social engineering
- Third-party service outages

Thank you for helping improve security.
